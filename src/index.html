About devfolio

ChatGPTを使い始めて9ヶ月。これからのWEBエンジニアの仕事とAIは切り離せないと確信している。
だからAIでサイトを作ってみた。

コンセプト
さて転職活動をするかという時にポートフォリオがないな、と思った。お客さんのサイトのソースコードを公開するわけにはいかないし、かと言って出力・公開されたものだけを見せるのも芸がない。
じゃあAI（LLM）に人間が伴走する形でポートフォリオサイトをプロジェクトとして作ってみようか。ということで作った。
開発フロー
詳細は「How I Built “devfolio"」をご覧いただくとして、単にHTMLを書かせるだけでは面白みに欠けるため、「できるだけ人間が手を動かさない」という大まかな方針を掲げた。
デザイン生成からプロトタイプ・指示書作成、逆コンパイルを用いた静的実装、ビルドやデプロイまでをLLMに任せることでLLMとの会話力も鍛えつつ、これからのLLM中心の開発フローを試すプロジェクトとして進めた。

Tech Notes

開発環境の効率化

手動ビルドプロセスが開発速度のボトルネックとなっていた課題を、Gulpによる自動化で解決。BrowserSyncによるライブリロードとStream形式でのスタイル注入により、リロード時間を大幅に短縮し、開発効率を向上させた実務経験

ビルド速度の改善

Webpackでのビルド時間が3分以上かかっていた課題に対し、Viteへの移行を実施。ビルド時間を30秒以下に短縮し、既存のWordPress開発環境（wp-env+Docker）への組み込みも成功。開発効率と開発体験を大幅に改善した実績

チーム開発環境の統一

メンバーごとに異なる開発環境による不具合や手順の違いが発生していた課題を、Dockerによる環境統一で解決。WSL2上での高速化によりWindows環境でも快適に動作し、オンボーディング時間の短縮にも成功した実務経験

既存テーマへの機能追加

既存のWordPressテーマにGutenbergブロック機能を追加する必要があった課題に対応。ACFブロックとの併用、動的ブロックの実装により、既存のデザインシステムを維持しながら柔軟なコンテンツ編集機能を実現した実務経験

検索順位改善の取り組み

検索エンジンでの表示順位が低かった課題に対し、構造化データ（JSON-LD）の実装とCore Web Vitalsの改善を実施。Google検索コンソールでの検証を経て、パフォーマンス指標と検索順位の向上を実現した改善実績

既存サイトのアクセシビリティ対応

既存サイトがWCAG 2.1ガイドラインに準拠していなかった課題に対応。WAI-ARIA属性の適切な追加、キーボード操作の検証、スクリーンリーダー対応などを実施し、アクセシビリティスコアを大幅に改善した実務経験

運用トラブル対応

メール配信問題（DKIM/DMARC設定）やサーバー設定の調査など、実務で遭遇した運用課題の解決経験。原因特定から設定変更まで、実際のトラブルシューティングの対応実績

レガシーコードのバグ修正

別の制作会社が作成したサイトのバグ修正対応。既存コードベースの調査、原因特定、影響範囲の確認から修正まで、レガシーコードを理解しつつ安全に修正する実務経験

How I Built "devfolio"

  1. 要件定義・技術選定

    ChatGPTでポートフォリオに必要な項目を洗い出し、要件を明確化しました。Cursor用の汎用指示書（mdc）を設置し、Cursorで開発計画とチェックリストなどを作成。以降はこのチェックリストに沿って開発を進めることで、一貫性と品質を担保しました。

  2. デザイン生成・プロトタイプ作成

    ChatGPTの作ったプロンプトを元にAnythingでReact+Vite+Tailwindのプロトタイプデザインを生成し、デザインを作成。デザイナーのポートフォリオサイトではないため、あまりここで時間はかけずに最大公約数的なデザインでほぼポン出しを採用しました。
    生成されたソースをダウンロードして中身を確認しますが、Reactに対して薄い理解しかないため、しばしフリーズの時間。

  3. 技術スタック移行・環境構築

    ChatGPTとCursor Proに相談しながらReact->HTML+CSS+JSへ変換し、HTMLからTailwindのClassを剥がしてPDFLOCSS形式へ移行。Gulp+SCSSの開発環境を構築し、BrowserSyncでのwatch、Stream形式でのスタイル注入を実装しました。
    ここでAnythingのデザイン・中身がざっくりしているために調整と加筆にかなり手間取りました。
    初手でもっとがっつり叩きまくってAnythingに作らせたほうが良かったのかもしれないとも思いましたが、デザイナーではないので言語化や指示にも限度があると思い、諦め。
    各所の文言はさすがにLLMに任せるにも限度があったので加筆修正多数。

  4. 実装・モジュール化

    この時点では公開したくない個人情報やコンテンツデータはダミーです。作ったHTMLをパーシャル（_header.ejs、_footer.ejs等）に分割し、GitHub公開時に公開したくない個人情報やコンテンツデータをCSVに外部化し、EJS->HTMLのビルド時に読み込む仕組みを構築しました。

  5. 品質保証・最適化

    WAI-ARIA属性の確認、キーボード操作の検証、Lighthouseによるアクセシビリティチェックを実施。パフォーマンス最適化としてクリティカルCSSの抽出、フォント最適化を実施。デプロイ前にはビルド確認、リンク切れチェック、クロスブラウザテストを徹底しました。

  6. CI/CD・デプロイ

    Cloudflare Pagesのデプロイ設定としてビルドコマンドを`gulp build`に設定。BASIC認証を設定し、本番環境への自動デプロイパイプラインを構築。GitHubへのプッシュと連動してデプロイが実行される仕組みを整備しました。

  7. リリース・運用確認

    本番環境での動作確認と最終調整を実施。実際のデプロイ後の動作を確認し、パフォーマンス指標やアクセシビリティスコアを検証。必要に応じて微調整を行い、安定した運用が可能な状態まで整備しました。
